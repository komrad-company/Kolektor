## Project
Catalogue of Vector.dev configurations for OCSF log normalisation.
Each config is a deployable data source, consumed by the Komrad stack and manageable via Kontrol.

## Target architecture
- `Kolektor` ships Vector and the local catalogue.
- `Kolektor-kontroler`, running in the Vector pod, reads `/etc/vector/catalog/index.json`, exposes the catalogue via gRPC, and applies active parsers.
- `Kontrol-api` orchestrates desired state and calls `Kolektor-kontroler`.
- No Rust REST API lives in this repo.

## Catalogue
- `catalog/*/*/manifest.yaml` contains human-readable metadata.
- `catalog/*/*/vector.toml` contains the Vector block applied by the controller.
- `catalog/index.json` is generated by `ci/catalog_index.py` and must remain in sync with the manifests.

## Conventions
- **Format**: TOML for Vector configs, VRL inline in transforms
- **Tests**: minimum 3 per source (nominal, optional_missing, malformed)
- **Test logs**: real raw samples, not invented
- **Runtime variables**: `${TENANT_ID}`, `${DATASOURCE_ID}`, `${LISTEN_PORT}`, `${QUICKWIT_ENDPOINT}`
- **Mandatory OCSF fields**: `class_uid`, `category_uid`, `severity_id`, `time`, `metadata`, `tenant_id`, `datasource_id`, `raw`
- **Commits**: `feat:`, `fix:`, `test:`, `ci:`, `docs:`
- **Language**: English for all docs and comments

## CI
- `python3 ci/catalog_index.py --check`
- `vector validate` on each `vector.toml` with dummy env vars injected by `ci/validate.sh`
- `vector test` on each source (vector.toml + tests/*.toml merged)
- `python3 ci/lint_vrl.py` (VRL antipatterns) + `python3 ci/validate_ocsf.py` (OCSF enum/category coherence)
- Markdown report as CI artifact
- CI image: `timberio/vector:0.55.0-debian`
- SAST: gitleaks + hadolint + grype (see `docs/CI.md`)

## Known pitfalls
- `vector test` requires tests to be in the same file or passed as arguments alongside the config
- Vector 0.54 `--no-environment` disables env-triggered reload but does NOT suppress `${VAR}` expansion: `ci/validate.sh` injects dummy vars accordingly
- VRL: no inline `if` inside object literals ‚Äî pre-compute values before assignment
- Vector `syslog` sources automatically parse the syslog header (timestamp, hostname, appname)
- In tests, use `type = "log"` with `insert_at` pointing to the transform
- When class_uid is dynamic (auditd, windows-evtx, crowdstrike), use a `route` transform + one sink per OCSF index
